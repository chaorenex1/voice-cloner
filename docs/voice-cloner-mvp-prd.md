# AI Voice Cloner MVP PRD

## 1. 产品概述

### 1.1 产品名称

- 工作名：Voice Cloner
- 产品定位：面向主播、游戏玩家、语聊用户与内容创作者的桌面端 AI 实时变声器

### 1.2 一句话定义

用户打开桌面应用，选择一个角色音色，即可把自己的麦克风声音实时转换为目标音色，并可在离线模式下将本地音频转换为目标声音导出。

### 1.3 MVP 目标

MVP 只解决一个核心问题：

- 让用户以尽量低的门槛获得“可实时使用”的 AI 变声体验

MVP 不追求：

- 任意名人声音克隆
- 开放式声音社区
- 浏览器端低延迟实时变声
- 多语言翻译配音
- 大规模云训练平台

### 1.4 产品形态

- 平台：多平台桌面端（Windows / macOS，后续评估 Linux）
- 形态：本地应用 + 云端推理
- 输出方式：
  - 实时虚拟麦克风输出
  - 本地文件离线导出

## 2. 背景与机会

当前语音 AI 产品很多，但用户在直播、连麦、游戏语音中真正需要的是：

- 延迟低，不影响交流
- 音色明显，但不失真
- 配置简单，不需要复杂训练
- 可直接接入 Discord、OBS、Zoom、游戏语音

MVP 的产品机会在于：

- 用“实时可用”替代“只会展示 demo 的克隆效果”
- 用“桌面链路稳定”切入高频使用场景
- 用“预置角色 + 简单自定义”建立初始留存

## 3. 目标用户

### 3.1 核心用户

1. 主播 / VTuber
   - 需要角色化声音
   - 需要 OBS、直播平台兼容

2. 游戏玩家 / 语聊用户
   - 需要实时连麦变声
   - 需要低延迟、切换方便

3. 短视频 / 二创创作者
   - 需要离线音频变声导出
   - 需要多个角色音色

### 3.2 次级用户

1. 轻度娱乐用户
   - 尝试不同声音做聊天和整活

2. 小团队内容工作室
   - 做角色配音、临时 demo 和短内容创作

## 4. 用户痛点

1. 传统变声器效果假，电子味重
2. 复杂 AI 语音产品门槛高，配置麻烦
3. 实时场景下延迟太大，无法自然对话
4. 很多产品只能生成文件，不能接虚拟麦克风
5. 用户不知道如何快速得到“好听、稳定、可用”的角色声音

## 5. MVP 成功定义

MVP 是否成立，取决于三件事：

1. 用户能在 5 分钟内完成首次实时变声
2. 用户能感知到明确音色变化且能正常连麦
3. 至少一部分用户愿意持续使用预置角色进行直播/聊天/录音

## 6. 产品目标与非目标

### 6.1 产品目标

- 提供可直接使用的实时变声能力
- 提供 3 到 5 个高可用预置角色音色
- 提供基本离线音频变声导出
- 提供稳定的设备设置、监听和虚拟麦克风输出
- 建立后续自定义声音训练与高级音色能力的扩展基础

### 6.2 非目标

- 不做开放声音市场
- 不做文本到语音主产品
- 不做复杂音频编辑器
- 不做移动端首发
- 不做大模型陪聊或语音助手产品

## 7. 核心使用场景

### 7.1 场景 A：游戏/语聊实时变声

用户打开应用，选择“少年音/御姐音/机械人/低沉男声”等角色，调好麦克风后直接输出到 Discord 或游戏语音。

### 7.2 场景 B：直播前预演

用户在直播前打开监听，边说边测试不同角色声音，找到合适音色后切到虚拟麦克风给 OBS 使用。

### 7.3 场景 C：离线音频变声

用户导入一段本地人声，选择角色音色后导出变声音频，用于短视频剪辑或 demo 配音。

## 8. 用户故事

1. 作为一个游戏玩家，我希望 3 分钟内完成设备设置并开始实时变声，这样我可以立刻在语音房使用
2. 作为一个主播，我希望实时监听自己的变声音频，这样我可以判断是否自然
3. 作为一个内容创作者，我希望上传一段音频并导出变声结果，这样我可以做角色配音素材
4. 作为一个新用户，我希望先用官方预置音色，而不是被迫先训练自己的声音
5. 作为一个普通用户，我希望应用能告诉我当前是否已正确输出到虚拟麦克风

## 9. MVP 范围

### 9.1 In Scope

#### 音色管理

- 监听开关
- 实时虚拟麦克风输出

#### 离线变声

- 导入单个本地音频文件
- 输入文本
- 选择目标角色进行转换
- 导出 WAV / MP3

#### 音色设计

- 语音识别
- 生成音色指令
- 生成参考音频和参考文本
- 保存到自定义音色库

#### 基础设置

- 输入/输出设备配置
- LLM配置
- 语音合成后端配置
- 语音识别后端配置

#### 本地配置

- 保存上次设备设置
- 保存上次角色设置

### 9.2 Out of Scope

- 用户上传样本训练专属声音

- 多轨编辑

- 社区分享和交易

## 10. 功能需求

### 10.1 功能模块一：音色管理

#### 功能描述

系统提供音色浏览、试听、切换与参数预设能力，作为实时变声与离线变声的统一入口。

#### 子功能

- 音色列表展示
- 音色试听样例
- 当前选中状态
- 快速切换目标角色
- 每个角色的参数预设加载

### 10.2 功能模块二：实时变声

#### 功能描述

用户通过麦克风说话，系统实时输出已转换的目标音色到监听与虚拟麦克风。

#### 子功能

- 开始/停止实时变声
- 输入设备选择
- 输出设备选择
- 监听开关
- 虚拟麦克风状态显示
- 延迟状态提示
- 角色切换
- 音高调节
- 音色强度调节
- 亮度调节
- 淡入淡出

#### 关键规则

- 未检测到输入设备时不可启动
- 首次启动需引导用户完成设备选择
- 切换角色时不中断应用，但允许短暂重新加载
- 若云端推理失败，需提示错误原因并自动回退到待机状态，而不是持续输出异常音频

### 10.3 功能模块三：离线音频变声

#### 功能描述

用户可上传单个本地音频文件，或直接输入文本，选择目标角色后生成并导出目标音频。

#### 子功能

- 文件导入
- 文本输入
- 角色选择
- 输入方式切换（音频 / 文本）
- 处理进度显示
- 导出文件

#### 关键规则

- MVP 支持单文件、单轨人声输入，或纯文本输入
- 支持常见格式：WAV、MP3、M4A
- 文件过长时给出限制提示
- 文本输入时需校验长度与非法空内容
- 文件上传、云端处理中、导出完成三个阶段必须有明确状态提示

### 10.4 功能模块四：音色设计

#### 功能描述

允许用户通过语音识别与 LLM 生成链路，将描述性需求转换为可复用的自定义音色条目。

#### 子功能

- 语音输入或文本输入设计需求
- 语音识别
- 生成音色指令
- 生成参考音频
- 生成参考文本
- 保存到自定义音色库

#### 关键规则

- 音色设计流程应可回看识别文本、生成指令与参考结果
- 生成失败时需区分是语音识别、LLM 生成还是语音合成阶段失败
- 保存到自定义音色库前，用户必须能够试听参考音频并确认
- MVP 阶段的音色设计以“指令生成 + 参考音频/文本 + 入库”为核心，不包含用户上传样本训练

### 10.5 功能模块五：设备与音频设置

#### 功能描述

提供音频设备与 AI 后端配置，确保用户能顺利连通真实设备、推理服务与虚拟输出。

#### 子功能

- 输入设备配置
- 输出设备配置
- LLM 配置
- 语音合成后端配置
- 语音识别后端配置

### 10.6 功能模块六：本地配置

#### 功能描述

保证用户重开应用后仍能保留偏好设置。

#### 子功能

- 保存上次设备设置
- 保存上次角色设置
- 保存最近离线导出记录

## 12. 关键流程

### 12.1 首次使用流程

1. 打开应用
2. 进入欢迎页
3. 选择输入/输出设备
4. 阅读虚拟麦克风说明
5. 进入首页
6. 选择一个角色
7. 点击开始实时变声
8. 打开监听确认效果

### 12.2 实时变声流程

1. 用户选择角色
2. 系统加载角色模型/参数
3. 用户点击开始
4. 系统检测麦克风音频流
5. 实时推理并输出
6. 用户在第三方 App 中选择虚拟麦克风

### 12.3 离线变声流程

1. 用户上传本地音频或输入文本
2. 系统校验文件格式与时长，或校验文本内容
3. 用户选择角色与参数
4. 点击开始转换
5. 系统处理并生成结果
6. 用户试听并导出

### 12.4 音色设计流程

1. 用户进入音色设计页
2. 用户录入语音或文本描述
3. 系统执行语音识别
4. 系统基于识别结果生成音色指令
5. 系统生成参考音频与参考文本
6. 用户试听并确认
7. 用户保存到自定义音色库

## 13. 关键体验要求

### 13.1 易用性

- 首次可用时间尽量低
- 核心操作不超过 3 次点击
- 角色切换逻辑清晰

### 13.2 反馈性

- 必须始终可见输入音量状态
- 必须明确显示当前是否在变声中
- 必须明确显示虚拟麦克风是否可用

### 13.3 可控性

- 用户能快速恢复推荐参数
- 用户能快速关闭监听，避免回声
- 用户能看懂错误，不被技术术语困住

## 14. 指标体系

### 14.1 北极星指标

- 周活跃实时变声用户数（Weekly Active Voice Changer Users）

### 14.2 核心产品指标

#### 激活指标

- 新用户首次启动后成功完成设备配置率
- 新用户首次启动后成功开启实时变声率
- 首次可用时间（从打开应用到首次成功输出变声音频）

#### 使用指标

- 日均实时变声启动次数
- 人均单次实时会话时长
- 角色切换次数
- 离线音频处理次数

#### 留存指标

- D1 留存
- D7 留存
- 首周完成 3 次以上实时会话的用户占比

#### 体验指标

- 实时链路端到端延迟
- 崩溃率
- 处理中断率
- 角色切换失败率
- 离线导出成功率

### 14.3 MVP 目标值

- 首次设备配置成功率 >= 85%
- 首次实时变声成功率 >= 75%
- 首次可用时间 <= 5 分钟
- 实时端到端延迟 <= 150ms
- 崩溃率 <= 2%
- 离线导出成功率 >= 95%
- D7 留存 >= 20%

## 15. 性能与质量指标

### 15.1 实时性能

- 音频采样率：优先 24k / 48k
- 端到端实时延迟：目标 <= 150ms，理想 <= 100ms
- 启动实时引擎时间：<= 3s
- 角色切换完成时间：<= 2s

### 15.2 音质指标

- 音色变化可感知
- 人声主语义清晰可懂
- 长句连续输出不明显破裂
- 无明显持续性爆音、卡顿、机械噪声

### 15.3 稳定性指标

- 单次连续运行 2 小时不崩溃
- 本地 CPU/内存峰值告警可感知
- 云端推理失败率可监控
- 异常时自动停机并提示

## 16. 风险与约束

### 16.1 技术风险

- 实时推理延迟超标
- 不同设备兼容性问题
- 虚拟麦克风接入复杂
- 低配机器性能不足

### 16.2 产品风险

- 用户期待“完全像某个真人”，但 MVP 无法满足
- 预置角色不够吸引人导致首日流失
- 参数过多导致新手困惑

### 16.3 合规风险

- 未授权的真人声音模仿
- 恶意伪造或诈骗用途
- 用户对生成内容归属理解不清

## 17. 安全与合规要求

- 明确禁止未授权真人声音克隆
- 产品内不提供公众人物音色预置
- 记录关键导出与高风险操作日志
- 后续需要为高级版本预留内容水印与追踪能力

## 18. MVP 里程碑

### M1：技术打底

- 完成桌面应用壳
- 完成麦克风采集与回放
- 完成基础实时推理链路

### M2：可用版本

- 接入预置角色
- 接入音色设计能力
- 完成虚拟麦克风输出
- 完成离线音频转换

### M3：封测版本

- 完成欢迎引导
- 完成设置页
- 完成崩溃/错误处理
- 完成基础埋点

### M4：公开 MVP

- 完成本地设置保存
- 完成体验优化
- 完成指标看板

## 19. 发布准入标准

满足以下条件才允许发布 MVP：

- 实时端到端延迟达到目标
- 主流程无 P0/P1 阻塞问题
- 至少 3 个预置角色达到可用质量
- Windows 与 macOS 主流设备完成基本兼容验证
- 离线导出主流程成功率达标

## 20. 后续演进方向

不属于 MVP，但应预留架构空间：

- 自定义声音训练
- 云端角色同步
- 多语言 dubbing
- 文本转目标角色声音
- 创作者市场
- 团队协作与 API
