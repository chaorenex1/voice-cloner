# Voice Cloner 应用整体布局设计

本文只定义 `voice-cloner` 的应用整体骨架，不展开任何具体页面内部设计。

目标只有两个：

- 固定左侧导航结构
- 明确右侧主内容区的整体承载方式

## 1. 布局目标

整体布局需要满足以下目标：

1. 用户打开应用后，能立即理解产品的主功能结构
2. 高频入口稳定可见，不需要通过多层跳转才能进入
3. 整个应用保持统一的桌面端骨架，而不是每个页面各自独立漂浮
4. 页面切换时只替换右侧内容，不改变整体导航结构

## 2. 整体骨架

应用采用固定的左右结构：

```text
+------------------------------------------------------------------------------------------------------+
| App Shell                                                                                            |
| +------------------------+-------------------------------------------------------------------------+ |
| | Left Sider             | Right Main Area                                                         | |
| | ---------------------- | ----------------------------------------------------------------------- | |
| | Logo / Product Name    | Main Content                                                            | |
| | 音色管理               | - 根据左侧当前导航展示对应页面                                          | |
| | 实时                   |                                                                         | |
| | 离线                   |                                                                         | |
| | 音色设计               |                                                                         | |
| | 设置                   |                                                                         | |
| | ---------------------- |                                                                         | |
| | 底部轻量状态区         |                                                                         | |
| +------------------------+-------------------------------------------------------------------------+ |
+------------------------------------------------------------------------------------------------------+
```

这个骨架表达的是：

- 左侧是固定导航区
- 右侧是统一主工作区
- 页面切换只发生在右侧内容区

## 3. 左侧 Sider

左侧导航从上到下固定为：

1. `音色管理`
2. `实时`
3. `离线`
4. `音色设计`
5. `设置`

这 5 个入口是一级导航，顺序固定，不做动态调整。

### 3.1 Sider 结构

左侧建议分成 3 段：

```text
+------------------------+
| Logo / 产品名          |
| Voice Cloner           |
|------------------------|
| 音色管理               |
| 实时                   |
| 离线                   |
| 音色设计               |
| 设置                   |
|------------------------|
| 底部轻量状态区         |
+------------------------+
```

说明：

- 顶部用于品牌识别
- 中部用于主导航
- 底部用于轻量级全局状态

### 3.2 Sider 原则

左侧 `sider` 应保持：

- 固定宽度
- 始终可见
- 只承载一级导航
- 不承载复杂操作面板
- 不因为页面切换而改变自身结构

## 4. 右侧主区域

右侧区域只保留一个统一容器：

- `Main Content`

### 4.1 Main Content

`Main Content` 是当前页面真正的工作区。

它的职责是：

- 根据左侧当前选中的导航项切换内容
- 承载该模块的完整页面
- 在不影响整体壳结构的前提下自由扩展

这里的关键原则是：

- 右侧不再额外拆出独立顶栏层
- 页面需要的标题、摘要或操作，由各自页面内部自行组织
- 应用层只统一外层骨架，不统一页面内部头部结构

## 5. 页面切换关系

左侧导航与右侧主内容区是一一对应关系：

- 点击 `音色管理`，右侧显示音色管理页面
- 点击 `实时`，右侧显示实时页面
- 点击 `离线`，右侧显示离线页面
- 点击 `音色设计`，右侧显示音色设计页面
- 点击 `设置`，右侧显示设置页面

也就是说：

- 左侧负责“去哪”
- 右侧负责“显示什么”

## 6. 骨架层级原则

为了让整个应用后续扩展稳定，建议遵守以下层级关系：

### 6.1 骨架稳定，内容切换

- 固定不变：左侧导航、右侧整体容器
- 按页面切换：右侧具体内容

### 6.2 导航清晰，内容独立

- 左侧只负责主模块切换
- 右侧每个页面独立承担自己的业务内容
- 不把页面内部细节重新塞回左侧导航

## 7. 结论

这份文档保留的整体骨架只有 4 个核心点：

1. 应用采用固定的左侧 `sider` + 右侧主内容区结构
2. 左侧导航顺序固定为：`音色管理 -> 实时 -> 离线 -> 音色设计 -> 设置`
3. 右侧整体结构只有一个 `Main Content`
4. 页面切换时只替换右侧内容，不改动整体壳结构

因此，这份文档的作用不是定义每个页面怎么画，而是定义整个应用的稳定外层框架。
